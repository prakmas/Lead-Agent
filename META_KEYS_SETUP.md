# Meta Keys Setup — Step by Step

This guide explains **exactly where to create each value** used by
`backend/.env` for WhatsApp Business, Instagram DM, and Facebook Messenger, with
direct links to the official Meta consoles and docs.

You only need these to send/receive real messages. For local development you can
leave them blank and use `AI_PROVIDER=mock` plus the simulator
(`POST /webhooks/simulate`) — see the main [README](README.md).

## Where each `.env` value comes from

| `.env` variable               | Where to get it                                                        |
| ----------------------------- | ---------------------------------------------------------------------- |
| `META_APP_SECRET`             | App Dashboard → **Settings → Basic → App Secret**                      |
| `META_VERIFY_TOKEN`           | You invent it (any long random string); paste the same value into Meta |
| `META_API_VERSION`            | Latest Graph API version, e.g. `v20.0` (keep the default unless told)  |
| `WHATSAPP_PHONE_NUMBER_ID`    | WhatsApp → **API Setup → Phone number ID**                            |
| `WHATSAPP_ACCESS_TOKEN`       | WhatsApp → **API Setup** (temp) or a System User token (permanent)     |
| `FACEBOOK_PAGE_ID`            | Your Facebook Page → **About**, or Messenger settings                  |
| `FACEBOOK_PAGE_ACCESS_TOKEN`  | Messenger → **Settings → Access Tokens → Generate Token**             |
| `INSTAGRAM_PAGE_ID`           | The Facebook Page connected to your professional IG account            |
| `INSTAGRAM_ACCESS_TOKEN`      | The connected Page's access token (same place as Messenger)            |

## Step 1 — Create a Meta app

1. Go to the **App Dashboard**: https://developers.facebook.com/apps
2. Click **Create App** → choose the **Business** type.
3. Add the products you need from the left sidebar:
   - **WhatsApp**
   - **Messenger**
   - **Instagram** (only if you'll use Instagram messaging)
4. Open **Settings → Basic** and copy the **App Secret**.

```bash
META_APP_SECRET=your_app_secret
```

> The app secret is used to verify the `X-Hub-Signature-256` HMAC on inbound
> webhooks. When it is set, this app rejects unsigned/forged webhook calls.

## Step 2 — Create a webhook verify token

This is **not** generated by Meta — you make up any long random string and paste
the **same value** into both Meta's webhook config and `.env`.

```bash
META_VERIFY_TOKEN=make-a-long-random-string
```

Generate one quickly:

```bash
node -e "console.log(require('crypto').randomBytes(24).toString('hex'))"
```

## Step 3 — Expose your backend over HTTPS

Meta requires an HTTPS callback URL. For local development, tunnel port 5000:

```bash
# ngrok
ngrok http 5000

# or Cloudflare Tunnel
cloudflared tunnel --url http://localhost:5000
```

Your webhook callback URL is the tunnel/domain URL plus `/webhooks/meta`:

```
https://your-tunnel-or-domain.com/webhooks/meta
```

## Step 4 — WhatsApp Business Cloud API

1. In the app, open **WhatsApp → API Setup**.
2. Copy the **Phone number ID** (a test number is provided by default).
3. Copy the temporary **access token**, or set up a permanent System User token
   (see Step 7).
4. Under **WhatsApp → Configuration → Webhook**, click **Edit** and enter your
   callback URL and the verify token from Step 2.
5. Subscribe to the **`messages`** webhook field.

```bash
WHATSAPP_PHONE_NUMBER_ID=your_phone_number_id
WHATSAPP_ACCESS_TOKEN=your_access_token
```

## Step 5 — Facebook Messenger

1. In the app, add **Messenger** and open its settings.
2. Under **Access Tokens**, connect a **Facebook Page** and **Generate Token**.
3. Copy the **Page ID** and the **Page Access Token**.
4. Under Messenger **Webhooks**, set the same callback URL + verify token.
5. Subscribe the Page to **message events** (`messages`, `messaging_postbacks`).

```bash
FACEBOOK_PAGE_ID=your_page_id
FACEBOOK_PAGE_ACCESS_TOKEN=your_page_access_token
```

## Step 6 — Instagram DM

Instagram messaging needs a **professional** Instagram account connected to a
Facebook Page.

1. Convert the Instagram account to a Professional (Business/Creator) account.
2. Connect it to a Facebook Page (Page Settings → Linked Accounts).
3. In the app, configure **Instagram → Messaging**.
4. Use the connected **Page's** access token (same value flow as Messenger).
5. Configure webhooks and subscribe to **Instagram message events**.

```bash
INSTAGRAM_PAGE_ID=your_connected_page_id
INSTAGRAM_ACCESS_TOKEN=your_page_access_token
```

## Step 7 — Permanent tokens for production

Temporary tokens expire (often within 24 hours). For production, create a
**System User** and generate a long-lived token:

1. Go to **Meta Business Settings**: https://business.facebook.com/settings
2. **Users → System Users → Add**, then **Generate New Token**.
3. Select your app and grant the needed permissions
   (`whatsapp_business_messaging`, `pages_messaging`, `instagram_manage_messages`,
   etc.).
4. Use this token for the relevant `*_ACCESS_TOKEN` values.

## Step 8 — Verify it works

1. Restart the backend after editing `.env`.
2. When you save the webhook in Meta, it calls `GET /webhooks/meta` — a correct
   `META_VERIFY_TOKEN` returns the challenge and Meta shows the webhook as
   verified.
3. Send a real DM to your WhatsApp test number / Page / Instagram account.
4. The conversation, lead, and any matches appear in the dashboard, and the
   customer receives an auto-reply.

## AI provider keys (optional)

The platform works without any paid AI using the built-in extractor:

```bash
AI_PROVIDER=mock
```

To switch providers, set `AI_PROVIDER` and the matching key:

```bash
# OpenAI — https://platform.openai.com/api-keys
AI_PROVIDER=openai
OPENAI_API_KEY=your_openai_key
OPENAI_MODEL=gpt-4.1-mini
```

```bash
# Anthropic Claude — https://console.anthropic.com/settings/keys
AI_PROVIDER=claude
CLAUDE_API_KEY=your_claude_key
CLAUDE_MODEL=claude-3-5-sonnet-latest
```

```bash
# Google Gemini — https://aistudio.google.com/app/apikey
AI_PROVIDER=gemini
GEMINI_API_KEY=your_gemini_key
GEMINI_MODEL=gemini-1.5-flash
```

## Security checklist

- Never commit a real `.env` file (it is git-ignored).
- Rotate any DB password, Meta token, or AI key pasted into chat, screenshots,
  tickets, or shared docs.
- Use separate development and production Meta apps where possible.
- Always serve webhooks over HTTPS and set `META_APP_SECRET`.
- Use a long random `JWT_SECRET`.
